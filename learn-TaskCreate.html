Use this skill whenever the user wants to translate a tech article, blog post, documentation, or text into Chinese, or says "translate this", "翻译", "/translate <url>", or wants to convert English tech content into readable Chinese. # Translate: Tech Content → Chinese HTML
7
8
Translate English tech blogs, docs, or text into Chinese. Output a single styled HTML file.
9
10
## Arguments
11
12
The user provides either:
13
- A **URL** to a tech blog or doc (e.g., `/translate https://example.com/article`)
14
- A **block of text** to translate directly
15
16
If a URL is given, fetch its content first. If neither is provided, ask the user what to translate.
17
18
## Workflow
19
20
### Step 1: Get the source content
21
22
**If URL:** Use WebFetch to retrieve the page content. Extract the article body — strip navigation, footers, ads, cookie banners. Keep headings, paragraphs, lists, tables, code blocks, and images.
23
24
**If text:** Use the text as-is.
25
26
### Step 2: Analyze before translating
27
28
Before translating, scan the source for:
29
30
1. **Ambiguous terms** — nouns used as subjects or objects with unclear referents. For example, "System" in an article about Claude agents could mean the OS, the AI system, or the orchestration layer. Identify every such term.
31
2. **Technical terms and keywords** — match against the term list below; note any new terms not in the list.
32
33
### Step 3: Translate to Chinese
34
35
Follow these style rules strictly — they define the voice and quality of the translation:
36
37
**精简短句，不说废话。**
38
39
- Use the shortest sentence that carries the full meaning. If a sentence can be split, split it.
40
- Classical Chinese (古文) phrasing is welcome when it makes the sentence tighter. For example, prefer "此法可行" over "这种方法是可以行得通的".
41
- Never pad sentences. No filler words. No rhetorical flourishes. No "值得注意的是", "众所周知", "不言而喻".
42
- Avoid passive voice where active voice is shorter. Prefer "模型生成回复" over "回复被模型所生成".
43
44
**译者注：消除歧义。**
45
46
- When a term in the original is ambiguous — especially nouns whose referent is unclear from context — add a translator note right after the term.
47
- Format: `<span class="translator-note">（译者注：指xxx）</span>`
48
- This is not optional. Ambiguity in the original must not silently pass into the translation. The reader deserves clarity the original author didn't provide.
49
50
**术语保留英文。**
51
52
- Technical terms and keywords stay in English. Wrap them in `<span class="term">...</span>`.
53
- If a term has a well-known Chinese equivalent that readers expect, use both: Chinese first, English in parentheses on first occurrence. E.g., "延迟（latency）". After that, use the English term alone.
54
- See the term list below. If you encounter a technical term not on the list, keep it in English and apply the same styling.
55
56
**保留结构。**
57
58
- Preserve headings, lists, tables, code blocks, blockquotes from the original.
59
- Code blocks must not be translated — keep them exactly as-is.
60
- If the original has section numbering, keep it.
61
62
### Step 4: Generate the HTML file
63
64
Create a single, self-contained HTML file.
65
66
**Structure:**
67
- `<h1>` for the article title (Chinese translation, with original English title in a subtitle)
68
- `<h2>` for sections, `<h3>` for subsections
69
- Tables as `<table>` with styled headers
70
- Code blocks in `<pre><code>` with syntax highlighting class if language is identifiable
71
- Blockquotes with `<blockquote>`
72
- Original source URL (if any) linked at the top
73
74
**Styling (inline `<style>` block):**
75
76
```css
77
body {
78
font-family: "Segoe UI", "Microsoft YaHei", "PingFang SC", sans-serif;
79
max-width: 860px;
80
margin: 0 auto;
81
padding: 2rem 1rem;
82
line-height: 1.85;
83
color: #1a1a1a;
84
background: #fff;
85
}
86
h1, h2, h3 { color: #1a56db; margin-top: 2rem; }
87
h1 { font-size: 1.8rem; border-bottom: 2px solid #1a56db; padding-bottom: 0.5rem; }
88
.subtitle { color: #666; font-size: 1rem; font-weight: normal; margin-top: 0.25rem; }
89
.source-link { color: #666; font-size: 0.9rem; margin-bottom: 2rem; display: block; }
90
.term { color: #1a56db; font-weight: bold; font-style: italic; }
91
.translator-note { color: #b45309; font-size: 0.9em; font-style: italic; }
92
pre { background: #f5f5f5; padding: 1rem; border-radius: 6px; overflow-x: auto; font-size: 0.9rem; }
93
code { font-family: Consolas, "Courier New", monospace; }
94
blockquote { border-left: 4px solid #1a56db; margin: 1rem 0; padding: 0.5rem 1rem; color: #555; background: #f8f9fa; }
95
table { border-collapse: collapse; width: 100%; margin: 1rem 0; }
96
th { background: #1a56db; color: #fff; padding: 0.6rem 1rem; text-align: left; }
97
td { padding: 0.6rem 1rem; border-bottom: 1px solid #e5e7eb; }
98
tr:nth-child(even) { background: #f8f9fa; }
99
img { max-width: 100%; height: auto; }
100
```
101
102
**File location:** Save as `<original-title-slug>_zh.html` in the current working directory.
103
104

106

107
start chrome "<path_to_html>"
108

109
110
